第一次通过ssh连接虚拟机，报错：Could not connect to '192.168.246.128' (port 22): Connection failed.
解决方案：
1、安装sshd：
apt install openssh-server
2、重启ssh服务
sudo service ssh restart 或
sudo /etc/init.d/ssh restart

这时，使用liwei/liwei可以登录了，登录之后可以su - root切换到root用户，但是还不能直接用root登录
解决方案：
编辑ssh的配置文件,命令：nano /etc/ssh/sshd_config
在Authentication部分，注释掉“PermitRootLogin without-password”
在Authentication部分，添加“PermitRootLogin yes”
使用“Ctrl+o”保存数据，使用“Ctrl+X”退出编辑器
重新启动ssh服务，命令：sudo service ssh restart

通过终端安装程序sudo apt-get install xxx时出错：
E: Could not get lock /var/lib/dpkg/lock - open (11: Resource temporarily unavailable)
E: Unable to lock the administration directory (/var/lib/dpkg/), is another process using it?
解决方案：
sudo rm /var/cache/apt/archives/lock
sudo rm /var/lib/dpkg/lock

--Ubuntu16.04在线安装Docker
参考官网：https://docs.docker.com/engine/installation/linux/ubuntu/#install-from-a-package
1、安装aufs驱动
sudo apt-get install \
    linux-image-extra-$(uname -r) \
    linux-image-extra-virtual
	
2、安装docker
sudo apt-get install docker-ce

--Ubuntu16.04通过安装包安装Docker（离线）
1、https://download.docker.com/linux/ubuntu/dists/下载安装包，选择Ubuntu版本
   我的是这个：https://download.docker.com/linux/ubuntu/dists/xenial/pool/stable/amd64/
   路径：/linux/ubuntu/dists/xenial/pool/stable/amd64/docker-ce_17.03.1~ce-0~ubuntu-xenial_amd64.deb 

2、安装docker
sudo dpkg -i docker-ce_17.03.1-ce-0-ubuntu-xenial_amd64.deb

3、验证安装是否成功
sudo docker run hello-world

--系统启动引导docker
sudo systemctl enable docker


--设置docker代理镜像仓库
获取镜像加速器地址（阿里云加速）

在不同的系统下面，配置加速器的方式有一些不同，所以我们介绍主要的几个操作系统的配置方法。 
关于加速器的地址，你只需要登录容器镜像服务的控制台，左侧的加速器帮助页面就会显示为你独立分配的加速地址。
https://cr.console.aliyun.com/#/accelerator
如何使用Docker加速器
针对Docker客户端版本大于1.10的用户

您可以通过修改daemon配置文件/etc/docker/daemon.json来使用加速器：

sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": ["https://4gmkkn6k.mirror.aliyuncs.com"]
}
EOF
sudo systemctl daemon-reload
sudo systemctl restart docker


重启docker服务
sudo service docker restart
